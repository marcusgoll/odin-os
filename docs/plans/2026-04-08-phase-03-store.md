# Phase 03 SQLite Store, Events, and Projections Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build the baseline SQLite runtime store, auditable event model, and replayable projection helpers for Odin OS.

**Architecture:** Phase 03 uses a hybrid transactional design. Current-state runtime tables provide direct lifecycle reads, while an append-only events table captures every important mutation in the same transaction so state survives restart and can be replayed into basic projections.

**Tech Stack:** Go 1.22, `database/sql`, SQLite, embedded SQL migrations, GNU Make
